
# Documentation structure

The separate markdown files have been split, named and structured in a logical way to support being combined into one chunk for rendering. The separation has been done to reduce the individual document size for complex sections.

The numbered prefixes on the file names can be interpreted as the chapter or section numbers of the content. The heading levels (H1, H2 etc) in the markdown are defined accordingly.


# Documentation specification

Use the separate `Documentation-specification.md` document to inform human and AI collaborators about the format and styles used in the documentation.


# Editing tools

The MacOS software application Obsidian is being used for writing this documentation. For reference, here are the settings in Obsidian:

**Essential configuration:**

- Options > Files and links > Use Wikilinks: ***Disable***
- Options > Files and links > Default location for new attachments: ***In sub folder under current folder***

**Optional (personal) preferences:**

- Options > Editor > Behaviour > Auto pair brackets: ***Disable***
- Options > Appearance > Base colour scheme: ***Light***
- To enable side-by-side editing:
	- Open .md file and switch to source mode
	- Right-click on the tab and select ***split right***
- Enable synchronized scrolling
	- Right-click on the tab and select ***Link with tab...***
